AI 帝国·数字中枢 MVP (v1.0) 技术架构与开发白皮书

1. 项目定位与 MVP 目标

本项目旨在通过 2D 俯视角像素沙盘（Phaser），将大模型多智能体（Multi-Agent）的工作流、思考状态及异常报错进行**“完全可视化”与“可干预化”**。
MVP (Demo v1.0) 的核心目标是向投资人/种子用户展示：系统如何将一句自然语言指令，转化为多个数字小人的协同工作，并在触发异常时通过“天牢”机制实现 Human-in-the-loop（人机协同拦截）。

2. 系统技术栈选型 (双核架构)

前端 UI 层 (文官系统)：Vite + React 18 + TailwindCSS

职责：负责页面布局、统帅控制台（侧边栏）、输入框、日志展示、以及拦截异常时的“批阅奏折”弹窗。

前端渲染层 (武将系统)：Phaser 3

职责：纯粹的 2D WebGL/Canvas 渲染引擎。负责加载地图 Tilemap、渲染小人（Sprite）的走动动画、处理寻路逻辑（A* Pathfinding）和坐标碰撞。

状态枢纽桥梁：Zustand

职责：解决 React 和 Phaser 的通信问题。React 往 Store 写入指令，Phaser 在 update() 循环中读取 Store 状态并驱动小人；反之亦然。

后端驱动层 (MVP 期 Mock)：Node.js 

职责：在 MVP 阶段，为了快速出效果，后端暂不接入真实的 LangGraph，而是写一个状态机脚本，通过 Server-Sent Events (SSE) 或 WebSocket 定时向下发伪造的 JSON 状态，模拟 Agent 的思考和报错延迟。

3. 免费 2D 像素美术资产 (Assets) 寻址指南

为了贴合“中式现代写实办公/红木家具”的设定，我们需要寻找 Top-down (俯视角)、32x32 或 16x16 像素的现代 RPG 素材。以下是极高性价比（甚至完全免费/CC0协议）的获取渠道：

3.1 核心素材库推荐平台

Itch.io (强烈推荐)

寻宝关键词：Modern Office Pixel Art (现代办公室), Top down RPG Asset, Furniture Tileset。

神级作者推荐：LimeZu。这位作者有大量现代写实风格的像素室内资产（包含真皮沙发、办公桌、电脑、甚至红木材质的变体），很多基础包是免费的（Modern interiors 系列），非常适合我们的“写实总控中心”设定。

OpenGameArt.org

全免费开源素材库。

寻宝关键词：搜索 LPC (Liberated Pixel Cup)。LPC 是一套非常标准的开源像素角色规范，里面有无数现代西装、极客装束的角色 Sprite Sheet（精灵图），且动画帧已经切好（上下左右行走），可以直接给我们的“六部小人”使用。

CraftPix.net

在它的 Freebies 分类下，有打包好的免费 2D 像素角色和办公室 UI 界面。

3.2 资产清单 (MVP 需要准备的素材)

地图 (Tilemap)：1张拼接好的办公室全景图（包含地板区分、太和殿红木桌、机柜、红白警戒线区域）。可以导出为 .png 背景 + .json 碰撞边界。

角色 (Sprite Sheet)：至少 2 个小人（如：穿长袍的丞相、穿格子衫的钦天监）。必须包含 Idle (站立) 和 Walk (四个方向行走) 的动画帧。

UI 皮肤：边框、滚动条的极简暗色调 UI 切图（可通过 TailwindCSS 纯代码实现，减少加载）。

4. 核心数据协议：前后端状态同步 (JSON Protocol)

这是整个帝国的“圣旨”协议。后端通过 WebSocket 向前端发送以下结构的 JSON，前端的 Phaser 引擎解析并执行动作。

4.1 协议数据结构设计

{
  "event_id": "msg_001",
  "timestamp": 1710000000,
  "type": "AGENT_STATE_UPDATE",
  "data": {
    "agent_id": "minister_of_works",      // 对应前端的“工部”小人
    "agent_name": "工部·研发 Agent",
    "status": "WORKING",                  // 状态枚举: IDLE, MOVING, WORKING, ERROR, JAILED
    "position": {
      "target_x": 450,                    // 目标坐标 X (如果是 MOVING 状态，小人会走向这里)
      "target_y": 300                     // 目标坐标 Y
    },
    "bubble_text": "正在检索开源模型库...",  // 小人头顶冒出的对话气泡内容
    "log_message": "[系统] 工部已接旨，正在调取 GitHub API" // 打印在右侧 React 控制台的日志
  }
}


4.2 特殊协议：触发“天牢”拦截机制 (Aha Moment)

当后端 Agent 检测到敏感词或死循环时，发送此拦截协议：

{
  "event_id": "err_002",
  "type": "CRITICAL_INTERCEPT",
  "data": {
    "agent_id": "search_agent",
    "status": "JAILED",                   // 前端侦测到此状态，瞬间把小人坐标设置为“天牢”区域
    "bubble_text": "⚠️ 触发风控，等待御批",
    "error_details": {
      "reason": "检测到未授权的本地文件读取操作 (Computer Use 越权)",
      "snippet": "fs.readFileSync('/etc/passwd')"
    },
    "requires_human": true                // 触发 React 弹出最高级别“批阅奏折”弹窗
  }
}


5. MVP 第一期开发里程碑 (Roadmap)

为了快速向投资人展示，建议按以下 3 个 Sprint（冲刺）执行：

Sprint 1：视觉与走动 (Visual & Movement) - 预计 3 天

完成 React + Phaser 的容器嵌套配置。

在 Itch.io 下载并加载一套免费的现代室内 Tileset。

在 Phaser 中实例化 2 个小人，并通过代码能控制他们从点 A 走向点 B，播放步行动画。

Sprint 2：状态流转与双向绑定 (The Bridge) - 预计 3 天

引入 Zustand 构建共享状态。

在 React 输入指令，将状态写入 Store，Phaser 读取 Store 并让“丞相”走到“工部”工位。

小人到达目的地后，头顶弹出气泡，右侧控制台输出相应日志。

Sprint 3：高光时刻“天牢审判” (Aha Moment) - 预计 2 天

写一个简单的 Node.js 脚本，模拟一次完整的流转过程：下达任务 -> 执行 -> 故意报错。

完善“瞬移到警戒区”的动画特效。

完成 React 的“御批拦截弹窗”，点击“释放”后，小人重新回到工位。

架构师结语：
第一版的代码不必苛求完美，但**“视觉隐喻”必须极度准确**。当小人被关进天牢的那一刻，就是投资人决定掏钱的那一刻。
