# 天命系统 (Mandate of Heaven) - 快速启动手册

> **一句话简介**：这是一个模拟古代朝廷决策的 AI Agent 系统。前端通过“拟定圣旨”下达指令，后端由“丞相”调度六部尚书（Agent）进行协同工作并回复。

---

## 🛠️ 1. 环境准备

确保你的电脑上安装了以下环境：
- **Node.js**: v18 或更高版本
- **包管理器**: npm 或 pnpm

---

## ⚙️ 2. 核心配置 (.env)

项目根目录下的 `.env` 文件是系统的动力源，请确保配置了正确的大模型 API。

**推荐配置 (DeepSeek)**:
```ini
# 核心 LLM 配置
OPENAI_API_KEY=sk-xxxxxx                  # 你的 API Key
OPENAI_BASE_URL=https://api.deepseek.com  # 模型厂商的 Base URL
OPENAI_MODEL=deepseek-chat                # 模型名称
```

> **注意**：本系统后端已移除对 OpenClaw 的依赖，完全由 `server/` 目录下的自研 Agent 框架驱动，**无需**安装或启动 Docker/OpenClaw Gateway。

---

## 🚀 3. 启动步骤 (双终端模式)

你需要同时启动**后端服务**和**前端界面**。请打开两个终端窗口分别运行：

### 🖥️ 终端 1：启动后端 (Backend)
后端负责 Agent 思考、工具调用和 Socket 通信。

```bash
# 进入项目根目录
cd /Users/hongyulin/Desktop/天命系统

# 启动后端服务
node server/index.js
```

**启动成功标志**：
看到类似以下的日志输出：
```
[Kernel] Registered Agent: minister
[System] Loaded 8 agents with OpenClaw architecture
Destiny System (Macro Architecture) running at http://localhost:3001
```

### 🎨 终端 2：启动前端 (Frontend)
前端负责展示朝廷画面和交互。

```bash
# 进入项目根目录
cd /Users/hongyulin/Desktop/天命系统

# 启动前端开发服务器
npm run dev
```

**启动成功标志**：
终端显示 `Local: http://localhost:5173/`。

---

## 📜 4. 开始上朝 (使用指南)

1.  **打开浏览器**：访问 [http://localhost:5173](http://localhost:5173)。
2.  **拟定圣旨**：
    - 点击页面底部的输入框（羽毛笔图标）。
    - 输入指令，例如：
        - `“查一下昨天的税收情况”`
        - `“让兵部尚书整顿边防”`
        - `“朕想知道国库还有多少银两”`
3.  **观察反馈**：
    - **丞相 (Minister)** 会收到指令，状态变为“正在思考...”。
    - 如果需要查数据，丞相会自动调度 **户部尚书** 或 **史官**。
    - 最终丞相会汇总信息，以“微臣遵旨”的口吻回复。

## ❓ 5. 常见问题 (FAQ)

**Q: 拟定圣旨后，没有任何反应，丞相不回话？**
- **A**: 99% 是因为后端服务没启动。请检查 **终端 1** 是否正在运行 `node server/index.js`。前端必须通过 WebSocket (端口 3001) 连接后端才能工作。

**Q: 提示 `EADDRINUSE: address already in use :::3001`？**
- **A**: 后端端口 3001 被占用了。
    - 解决方法：`lsof -i :3001` 找到 PID，然后 `kill -9 <PID>` 杀掉旧进程，再重新启动。

**Q: Agent 回复报错或乱码？**
- **A**: 检查 `.env` 中的 `OPENAI_API_KEY` 是否有效，或者余额是否充足。
