# OpenClaw 子代理 (Sub-agent) 系统深度解析

OpenClaw 的核心强大之处在于其分层的子代理系统，允许 Agent 动态创建（Spawn）子 Agent 来并行处理任务。本文档详细解析该系统的实现原理。

## 1. 核心架构

子代理系统主要由以下三个模块组成：

1.  **Spawn (孵化)**: 负责创建新的 Agent 会话，注入上下文。
    - 源码: `src/agents/subagent-spawn.ts`
2.  **Registry (注册表)**: 负责追踪子 Agent 的生命周期、状态和持久化。
    - 源码: `src/agents/subagent-registry.ts`
3.  **Auto-Announce (自动通告)**: 负责将子 Agent 的执行结果“推送”回父 Agent。
    - 源码: `src/agents/subagent-announce.ts`

## 2. 孵化流程 (`sessions_spawn`)

当父 Agent 调用 `sessions_spawn` 工具时，系统执行以下步骤：

### 2.1 校验与准备
- **深度检查**: 检查当前调用深度 (`callerDepth`) 是否超过 `maxSpawnDepth` (默认通常为 3-5 层)。
- **数量限制**: 检查当前 Agent 已活跃的子 Agent 数量是否超过 `maxChildrenPerAgent`。
- **Agent ID 校验**: 确保目标 `agentId` 有效且在允许列表 (`allowAgents`) 中。

### 2.2 会话创建
- 生成唯一的 **Session Key**: 格式为 `agent:<targetAgentId>:subagent:<UUID>`。
- 继承或指定 **Workspace**: 子 Agent 可以继承父 Agent 的工作目录或使用独立的临时目录。

### 2.3 上下文注入 (System Prompt Injection)
系统会动态构建子 Agent 的 System Prompt，注入以下关键指令：
- **角色定义**: "You are a subagent spawned by..."
- **任务聚焦**: "Complete this task. That's your entire purpose."
- **禁止轮询**: "Auto-announce is push-based... do NOT call sessions_list... Wait for completion events."
- **父子关系**: 明确它是为了完成特定任务而生，任务完成后即被销毁（Ephemeral）。

### 2.4 启动执行
通过 Gateway 的 `agent` 方法发送初始任务消息，触发目标 Agent 开始思考和执行。

## 3. 注册表管理 (Registry)

`SubagentRegistry` 是子代理系统的“户籍管理中心”。

- **状态追踪**: 维护一个 `subagentRuns` 映射表，记录每个运行实例的 `runId`, `childSessionKey`, `requesterSessionKey` (父会话), `status` 等。
- **持久化**: 状态会定期保存到磁盘，确保系统重启后能恢复对子 Agent 的管理。
- **生命周期事件**: 监听 `lifecycle` 事件流（Start, End, Error），一旦子 Agent 结束运行，立即触发后续流程。

## 4. 自动通告机制 (Auto-Announce)

这是 OpenClaw 与传统 Multi-Agent 系统最大的不同点：**Push over Poll** (推送优于轮询)。

### 4.1 工作原理
父 Agent 不需要循环调用 `check_status`。当子 Agent 完成任务（输出最后一条消息并停止生成）时，`SubagentRegistry` 会捕获该事件。

### 4.2 结果捕获
`captureSubagentCompletionReply` 函数会读取子 Agent 的最后一条 Assistant 回复或 Tool Result。

### 4.3 消息回传
系统将结果封装成一个标准的用户消息，发送给父 Agent：
```text
[Subagent Task]: <Task Description>
Status: completed successfully
Result: <Child Agent's Output>
```

### 4.4 唤醒父 Agent
收到这条“伪造”的用户消息后，父 Agent 会被自动唤醒，将子 Agent 的结果纳入上下文，继续执行后续逻辑。

## 5. 工具集 (Tools)

OpenClaw 为 Agent 提供了以下核心工具来管理子系统：

- **`sessions_spawn`**: 创建子 Agent。
  - 参数: `agentId`, `task`, `model` (可选), `context` (可选)。
- **`subagents`**: 管理子 Agent。
  - `list`: 列出当前活跃的子 Agent。
  - `kill`: 强制终止某个子 Agent。
  - `prune`: 清理已完成的记录。

## 6. 总结

OpenClaw 的子代理系统设计极其精巧，通过 **"Spawn -> Execute -> Auto-Announce -> Wake Parent"** 的闭环，实现了高效的异步任务分发。它避免了父 Agent 消耗 Token 去轮询状态，同时通过 System Prompt 严格约束子 Agent 的行为，保证了任务执行的专注度。
